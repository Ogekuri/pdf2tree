## Quick Start

### Prerequisites

- Use supported environments: `linux`
- Install the `uv` tool from: [https://docs.astral.sh/uv/getting-started/installation/](https://docs.astral.sh/uv/getting-started/installation/)


### Run pdf2tree/req live with uvx
```bash
uvx --from git+https://github.com/Ogekuri/pdf2tree.git pdf2tree
```
### Install/Uninstall pdf2tree/req with uv

#### Install
```bash
uv tool install pdf2tree --force --from git+https://github.com/Ogekuri/pdf2tree.git
```

#### Uninstall
```bash
uv tool uninstall pdf2tree
```

### Usage hints
The CLI automatically switches to a deterministic **test mode** whenever it detects `PYTEST_CURRENT_TEST` or the environment variable `PDF2TREE_TEST_MODE=1`. In this mode no network requests or Pix2Tex invocations are performed; instead the following canned responses drive the pipeline so that unit tests remain reproducible:

- `PDF2TREE_TEST_PIX2TEX_FORMULA`: overrides the LaTeX formula embedded before equation images (defaults to a valid integral expression that passes the validator in `latex.py`).
- `PDF2TREE_TEST_GEMINI_IMAGE_ANNOTATION`: overrides the Markdown snippet injected below standard images.
- `PDF2TREE_TEST_GEMINI_EQUATION_ANNOTATION`: overrides the Markdown snippet injected below equation images; when unset it automatically includes the canned Pix2Tex formula rendered as MathJax.

These variables affect only the deterministic test path; normal executions continue to call Pix2Tex and Gemini as configured.
